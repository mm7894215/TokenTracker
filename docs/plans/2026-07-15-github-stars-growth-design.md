# GitHub Stars Growth Design

Date: 2026-07-15

## Goal

Grow `mm7894215/TokenTracker` GitHub Stars through legitimate product value,
clear positioning, discoverability, community participation, and measurable
product-led sharing. Do not buy stars, automate fake accounts, mass-DM people,
or hide self-promotion.

## Baseline

Snapshot taken on 2026-07-15:

- 990 Stars and 98 forks.
- 298 new Stars in the last 30 days, 195 in 14 days, and 64 in 7 days.
- 1,516 unique GitHub visitors and 1,121 unique repository-home visitors in
  the last 14 days.
- 699 unique clones in the last 14 days.
- 14,300 npm downloads in the latest available 30-day window.
- Strongest visible external referrers: Google (327 unique visitors) and
  linux.do (87 unique visitors).
- Existing distribution proof: featured in 阮一峰科技爱好者周刊 #393 and
  merged into `jaywcjlove/awesome-mac`.
- Two relevant, mergeable awesome-list PRs remain open.

The repository is already converting real interest. The immediate problem is
not lack of product depth; it is inconsistent discovery metadata, an
undifferentiated "many tools" message, and underused visual and sharing loops.

## Positioning

Lead with the product experience competitors do not currently advertise:

> The AI usage tracker that lives with you: a desktop pet powered by your real
> token activity, four desktop widgets, and achievements — backed by one
> local-first dashboard for 27 coding tools on macOS and Windows.

The desktop pet is the visual hook, widgets are the everyday utility, and
achievements turn tracking into a progress loop users can share. Provider
count, privacy, and accurate deduplication are the proof underneath that hook.
Search results now contain several tools claiming 30+ integrations, while the
major competitors reviewed do not advertise this pet + widgets + achievements
combination. TokenTracker should own the "AI coding companion" position.

## Growth System

### 1. Repair discovery and conversion

- Keep the README, GitHub description, website structured data, static
  AI-readable content, `llms.txt`, and marketing tool wall synchronized.
- Put the strongest proof near the first action: local-first, native macOS and
  Windows, 27 tools, one-command install, and no prompt collection.
- Lead launch visuals with the animated desktop pet, then show the four widgets
  and achievement unlocks before the conventional dashboard screenshots.
- Preserve a visible but non-blocking Star action on the website and in the
  installed dashboard.
- Use shareable public profiles and README badges as the durable backlink loop.

### 2. Run a milestone launch, not a generic announcement

- Use the 1,000-Star milestone as a concrete story: what was learned while
  turning raw token logs into a desktop companion, four widgets, and an
  achievement system, plus what was learned normalizing 27 incompatible tools.
- Publish in waves so every community gets a native post rather than copied
  spam: developer communities first, macOS/Windows communities second, and
  observability/LLMOps lists third.
- Always disclose that the poster is the maintainer and engage with replies.

### 3. Compound through product-led sharing

- Make the existing public profile and README badge easier to discover after a
  user has signed in and synced successfully.
- Add share moments for newly unlocked achievements and distinctive pet states;
  shared media must visibly identify TokenTracker and link back to the project.
- Add source attribution to outbound links and share URLs so website, app,
  profile, badge, and community traffic can be compared.
- Ask for a Star only after the user has received value; never gate features or
  interrupt first run.

## Targets

- Immediate: cross 1,000 Stars and capture the milestone assets.
- 30-day base target: 1,300 Stars; stretch target: 1,500.
- 90-day target: 2,000 Stars with at least half of new Stars coming from
  durable search, listings, badges, and user sharing rather than one launch.

Track weekly: total Stars, net Stars/day (7d and 30d), repository unique
visitors, visitor-to-Star proxy, unique clones, npm downloads, top referrers,
badge/profile shares, and outbound GitHub clicks.

## Execution Order

1. Fix stale 13/25-tool discovery metadata and the incomplete marketing wall.
2. Prepare the 1K launch kit with channel-specific copy and assets.
3. Follow up existing mergeable awesome-list submissions; submit only to
   curated lists where TokenTracker clearly belongs.
4. Instrument and improve the existing profile/badge sharing path.
5. Review weekly data, keep winning sources, and stop channels that produce
   clicks without installs, Stars, or useful community discussion.

## Guardrails

- No purchased or exchanged Stars.
- No bots, sock-puppets, vote rings, or misleading claims.
- No unsolicited bulk DMs or issue/PR spam.
- No fake comparison numbers or unverified "best" claims.
- Respect every community's self-promotion rules and maintainer decisions.
